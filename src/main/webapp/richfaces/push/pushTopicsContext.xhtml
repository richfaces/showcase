Push can be controlled by programmatic API. However durable topic still needs to be created and registered in RichFaces and Java Messaging Service.

Programmatic API consists from TopicKey which addresses the topic and TopicsContext used for publishing messages.

For the purpose of observing events from the server, client creates pernament connection to the server either by WebSocket connection or long-polling method. Pernament connections are managed by Atmosphere Framework.

In this example, message doesn't contain any data - it instructs client to rerender part of the page, in this case it forces client to load new unique ID.
